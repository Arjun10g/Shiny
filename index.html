Learn Shiny from Scratch

Setting up Shiny

Shiny is an R package that allows users to create interactive web applications directly from R. It is a powerful tool for data visualization, exploration, and communication, enabling users to build interactive dashboards, web applications, and reports without needing to learn web development languages like HTML, CSS, or JavaScript. Shiny leverages the R programming language, making it easy for R users to create interactive web applications using their existing knowledge and skills.

  1. Install R and RStudio:
    • Download and install the latest version of R from CRAN.
    • Download and install RStudio from RStudio's website to make managing your R projects easier.
  2. Install the Shiny package in R:
    • Open RStudio and run the command 
      install.packages("shiny")
      in the console to install Shiny.
  3. Create a new R script or R Markdown file:
    • In RStudio, go to File > New File > R Script for a basic script or R Markdown for a more complex application with narrative text and outputs.
  4. Load the Shiny library:
    • At the beginning of your script, add 
      library(shiny)
      to load the package.
  5. Define UI (User Interface):
    • Once you have set up a shiny script and set a directory for your app, it's time to start. We begin by establishing the basics and defining a user-interface. A user interface is what the user interacts with, and can be defined in a multitude of ways.
    • This section is primarily working with defining the html code behind the scenes and giving users the framework to add inputs and a field to show the output.
  6. Define server logic:
    • Define the server logic that will process user inputs and generate outputs.
    • This section is primarily working with the R code that will be executed when the user interacts with the UI.
    • This function/logic is primarily working with JS (Javascript) to accept user inputs and return some form of output.
  7. Run the app:
    • Use the following code: 
      shinyApp(ui = ui, server = server)
      at the end of your script to run the app directly from RStudio.
    • Alternatively, you can use the 'Run App' button in RStudio when your script is open.
  8. Deploy the app (optional):
    • Deploy your app online using services like shinyapps.io or your own Shiny Server.

The user interface (UI) is the visual part of a Shiny app that users interact with. It defines the layout and appearance of the app, including input controls like sliders, text boxes, and buttons, as well as output elements like plots, tables, and text.

Reactivity

Function Description Example Code
reactive() Creates a reactive expression that updates when its reactive dependencies change. 

        reactiveText <- reactive({
          paste("You entered:", input$text)
        })
        output$text <- renderText({
          reactiveText()
        })
observe() Creates an observer that runs whenever its reactive dependencies change, without returning a value. 

        observe({
          print(input$text)
        })
observeEvent() Creates an observer that triggers only when a specific event occurs, e.g., a button press. 

        observeEvent(input$btn, {
          print("Button clicked")
        })
reactiveValues() Creates a list of reactive values that can store multiple reactive states. 

        rv <- reactiveValues(count = 0, msg = "Hello")
        observeEvent(input$btn, {
          rv$count <- rv$count + 1
          rv$msg <- paste("You clicked", rv$count, "times")
        })
        output$text <- renderText({
          rv$msg
        })
reactiveVal() A simpler version of reactiveValues() for handling a single reactive value. 

        count <- reactiveVal(0)
        observeEvent(input$btn, {
          count(count() + 1)
        })
        output$text <- renderText({
          paste("Count is", count())
        })
eventReactive() Creates a reactive expression that updates only when a specific event occurs. 

        result <- eventReactive(input$btn, {
          paste("You clicked the button", input$btn, "times")
        })
        output$text <- renderText({
          result()
        })
isolate() Accesses a reactive value without triggering reactivity. 

        output$text <- renderText({
          isolate({
            paste("You entered:", input$text)
          })
        })
invalidateLater() Sets up a reactive expression to re-evaluate after a specific time interval. 

        output$currentTime <- renderText({
          invalidateLater(1000)  # Re-evaluates every second
          paste("Time:", Sys.time())
        })
req() Ensures that a reactive expression only proceeds if a required condition is met. 

        output$text <- renderText({
          req(input$text != "")
          paste("You entered:", input$text)
        })
freezeReactiveValue() Prevents reactivity from being triggered for a specific input or value. 

        freezeReactiveValue(input, "textInput")
bindCache() Caches the result of a reactive expression to avoid unnecessary re-computation. 

        output$plot <- renderPlot({
          bindCache({
            plot(cars)
          }, input$dataset)
        })
debounce() Delays a reactive expression’s update until the input stops changing for a certain period. 

        debouncedText <- debounce(reactive(input$text), 500)
        output$text <- renderText({
          paste("You entered:", debouncedText())
        })
throttle() Throttles a reactive expression, causing it to update at a fixed interval. 

        throttledText <- throttle(reactive(input$text), 1000)
        output$text <- renderText({
          paste("You typed:", throttledText())
        })

User Feedback

Function Description Example Code
validate() Displays a custom validation message to users if a condition is not met. Stops the execution of the current reactive context. 

        validate(need(input$age > 18, "You must be over 18 years old"))
need() A helper function used with validate() to check if a condition is true. If false, it shows the provided error message. 

        validate(need(input$age, "Age is required"))
req() Ensures that an input or condition is available before proceeding. It pauses reactivity until the required input is provided. 

        req(input$name)
tryCatch() Provides error handling by executing code within the try block and catching any errors that occur, allowing you to show error messages. 

        output$result <- renderText({
          tryCatch({
            if (input$number < 0) stop("Number must be positive")
            paste("The number is", input$number)
          }, error = function(e) {
            paste("Error:", e$message)
          })
        })
withProgress() Displays a progress bar to indicate the ongoing progress of a long-running operation. 

        withProgress(message = "Calculating...", value = 0, {
          for (i in 1:10) {
            incProgress(1/10)
            Sys.sleep(0.1)
          }
        })
progressBar() A custom progress bar function, often from external packages like shinyWidgets, to give users feedback on app loading or processing. 

        progressBar(id = "pb", value = 50, total = 100)
isolate() Prevents a reactive expression from being invalidated by inputs, making the expression non-reactive until explicitly triggered. 

        output$result <- renderText({
          isolate({
            paste("You entered:", input$name)
          })
        })
showNotification() Displays a temporary notification message to provide feedback to users. Can be used to show success, warning, or error messages. 

        showNotification("Data saved successfully!", type = "message")
removeNotification() Removes an active notification that was previously shown with showNotification(). 

        id <- showNotification("Processing...", duration = NULL)
        removeNotification(id)
shiny::onStop() Specifies code to run when the session is about to end, such as cleanup tasks or saving user input. 

        session$onStop(function() {
          saveData(input$dataset)
        })
validateCssUnit() Validates a CSS unit (e.g., "10px", "20%") for proper formatting before using it in UI elements. 

        validateCssUnit("50px")  # Valid
        validateCssUnit("abc")   # Error
observe() Watches for changes in reactive inputs but does not return a value. Useful for triggering side effects like logging or UI updates. 

        observe({
          if (input$submit) {
            cat("Submit button clicked")
          }
        })
observeEvent() Similar to observe() but triggered by a specific event, such as a button click or change in a reactive input. 

        observeEvent(input$submit, {
          cat("Submit clicked. Name:", input$name)
        })
freezeReactiveValue() Temporarily prevents an input from triggering reactive changes, often used when updating multiple inputs simultaneously. 

        freezeReactiveValue(input, "id")

First App

Here is a simple example of a Shiny app that takes several user inputs and outputs descriptives and a graph:

          ui <- fluidPage(
                titlePanel('Normal Distribution'),
                sidebarLayout(
                  sidebarPanel(tabsetPanel(
                    tabPanel('Distribution',
                            sliderInput('n','Sample Size',value = 100,min = 10,max = 1000,step = 10),
                            sliderInput('mean','Mean',value = 0,min = -10,max = 10,step = 0.1),
                            sliderInput('sd','Standard Deviation',value = 1,min = 0.1,max = 10,step = 0.1)
                            ),
                    tabPanel('Graph Settings',
                            selectInput('color','Pick a Color',choices = colours()),
                            selectInput('fill','Fill Color',choices = colours()),
                            sliderInput('alpha', 'Transparency',value = 0.5,min = 0,max = 1,step = 0.1)
                            )
                            ),
                    actionButton('gen', 'Generate'),width = 12),
                  mainPanel(
                    tabsetPanel(
                      tabPanel('Graph',plotOutput('a')),
                      tabPanel('Descriptives',verbatimTextOutput('b'))
                    ),width = 12)
                )
              )

              server <- function(input,output,session){
                observeEvent(
                  input$gen,{
                    data <- rnorm(input$n,input$mean,input$sd)
                    color = input$color
                    output$a <- renderPlot({
                      hist(data,col = color,breaks = 30,main = 'Normal Distribution')
                    })
                    output$b <- renderText({
                      data <- rnorm(input$n,input$mean,input$sd)
                      paste('Mean:',mean(data),'\n',
                            'Standard Deviation:',sd(data),'\n')
                    })
                  }
                )
              }

              shinyApp(ui = ui, server = server)

This app allows users to specify the sample size, mean, and standard deviation of a normal distribution, and customize the appearance of the histogram. When the "Generate" button is clicked, the app generates a random sample, displays the histogram, and outputs the mean and standard deviation of the sample.

  1. Sample Size: The number of data points in the sample.
  2. Mean: The average value of the normal distribution.
  3. Standard Deviation: The spread or variability of the data.
  4. Color: The color of the histogram bars.
  5. Fill Color: The color to fill the histogram bars.
  6. Transparency: The transparency level of the histogram bars.

When the "Generate" button is clicked, the app generates a random sample of the specified size and parameters, creates a histogram with the selected colors and transparency, and displays the mean and standard deviation of the sample.

This example demonstrates how to create a simple Shiny app with user inputs, reactive outputs, and interactivity using the shiny package in R.

Adding Validity Checks

It is important to add validity checks to ensure that the user inputs are within acceptable ranges or meet specific criteria. This helps prevent errors and ensures that the app functions correctly.

          ui <- fluidPage(
                  titlePanel('Normal Distribution'),
                  sidebarLayout(
                    sidebarPanel(
                      tabsetPanel(
                        tabPanel('Distribution',
                                sliderInput('n', 'Sample Size', value = 100, min = 10, max = 1000, step = 10),
                                sliderInput('mean', 'Mean', value = 0, min = -10, max = 10, step = 0.1),
                                sliderInput('sd', 'Standard Deviation', value = 1, min = 0.1, max = 10, step = 0.1)
                        ),
                        tabPanel('Graph Settings',
                                selectInput('color', 'Pick a Color', choices = colours()),
                                selectInput('fill', 'Fill Color', choices = colours()),
                                sliderInput('alpha', 'Transparency', value = 0.5, min = 0, max = 1, step = 0.1)
                        )
                      ),
                      actionButton('gen', 'Generate'), width = 12
                    ),
                    mainPanel(
                      tabsetPanel(
                        tabPanel('Graph', plotOutput('a')),
                        tabPanel('Descriptives', verbatimTextOutput('b'))
                      ), width = 12
                    )
                  )
                )

                server <- function(input, output, session) {
                  observeEvent(input$gen, {
                    # Validity checks
                    validate(
                      need(input$n > 0, "Sample size must be greater than 0."),
                      need(input$sd > 0, "Standard deviation must be greater than 0.")
                    )
                    
                    data <- rnorm(input$n, input$mean, input$sd)
                    color <- input$color
                    
                    output$a <- renderPlot({
                      hist(data, col = color, breaks = 30, main = 'Normal Distribution')
                    })
                    
                    output$b <- renderText({
                      validate(
                        need(input$n > 0, "Sample size must be greater than 0."),
                        need(input$sd > 0, "Standard deviation must be greater than 0.")
                      )
                      
                      # Generate new data for descriptive statistics
                      data <- rnorm(input$n, input$mean, input$sd)
                      paste('\n','Mean:', round(mean(data), 2), '\n',
                            'Standard Deviation:', round(sd(data), 2), '\n')
                    })
                  })
                }

                shinyApp(ui = ui, server = server)

In this updated version of the app, we have added validity checks to ensure that the sample size and standard deviation are greater than zero. If these conditions are not met, the app will display an error message and prevent the generation of the histogram and descriptive statistics.

Using Modals

          server <- function(input, output, session) {
            observeEvent(input$gen, {
              
              # Validity checks with pop-up alerts
              if (input$n <= 0) {
                showModal(modalDialog(
                  title = "Input Error",
                  "Sample size must be greater than 0.",
                  easyClose = TRUE,
                  footer = NULL
                ))
                return(NULL)
              }
              
              if (input$sd <= 0) {
                showModal(modalDialog(
                  title = "Input Error",
                  "Standard deviation must be greater than 0.",
                  easyClose = TRUE,
                  footer = NULL
                ))
                return(NULL)
              }
              
              data <- rnorm(input$n, input$mean, input$sd)
              color <- input$color
              
              output$a <- renderPlot({
                hist(data, col = color, breaks = 30, main = 'Normal Distribution')
              })
              
              output$b <- renderText({
                # Generate new data for descriptive statistics
                data <- rnorm(input$n, input$mean, input$sd)
                paste('Mean:', round(mean(data), 2), '\n',
                      'Standard Deviation:', round(sd(data), 2), '\n',
                      'Skewness:', round(skewness(data), 2), '\n',
                      'Kurtosis:', round(kurtosis(data), 2))
              })
            })
          }
          shinyApp(ui = ui, server = server)

In this version of the app, we have replaced the validate() function with modal dialogs that display pop-up alerts when the sample size or standard deviation is less than or equal to zero. This provides a more user-friendly way to communicate errors and prevent the app from proceeding with invalid inputs.
